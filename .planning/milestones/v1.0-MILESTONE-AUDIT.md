## Tech Debt

### 1. SESS-02 Requirement/Code Mismatch (Low)
**Details:** SESS-02 specifies `SameSite=Strict`. Code uses `SameSite=Lax` at `apis/dashboard/index.ts` lines 593, 744, 768, 953. `Lax` is required because Stripe Checkout redirects would drop a `Strict` cookie, breaking the post-payment flow. Phase 9 VERIFICATION.md incorrectly asserts "Strict".
**Action:** Update SESS-02 requirement text to `SameSite=Lax`. Correct Phase 9 VERIFICATION.md assertion.

### 2. Phase 1 SUMMARY Frontmatter Gaps (Low)
**Details:** Plans 01-01 and 01-02 lack `requirements-completed` YAML frontmatter. 7 requirement IDs not listed in any SUMMARY but all confirmed SATISFIED in VERIFICATION.
**Action:** Add frontmatter to 01-01-SUMMARY.md (INFRA-01, INFRA-02) and 01-02-SUMMARY.md (SESS-01, SESS-07, BILL-06, KEY-02, KEY-04).

### 3. Revenue Table User Attribution (Low)
**Details:** `revenue` table lacks `user_id` and `api_key_id` columns. API key revenue uses `network='credits'` for aggregate counting but can't be attributed to individual users without joining through `credit_transactions`.
**Action:** Consider adding columns in v2 migration for analytics dashboard.

### 4. Human Verification Items (Info)
**Details:** Two deployment-dependent Phase 11 checks unconfirmed: (a) API key paid request logs paid=true at runtime, (b) Caddy header stripping effective after reload.

### 5. INT-08 Multi-Phase Claims (Info)
**Details:** INT-08 claimed by 5 plan summaries across phases 7, 9, 11. Reflects incremental implementation, not an error.

**Total: 5 items (2 low-severity doc fixes, 1 low-severity analytics gap, 2 informational)**
